# 画像・視認性設計案

対象タイトル: AIで商談準備を速くするなら、最初に見るべき顧客情報は何か

## 1. 記事構成の想定

1. 導入: 商談準備の遅さは「情報量不足」ではなく「見る順番の曖昧さ」から起きる。
2. 問題提起: AIに情報を集めさせても、顧客理解の軸がないと要約の羅列になる。
3. 最初に見る情報: 企業属性、直近の変化、接点履歴、課題仮説、意思決定構造の順で見る。
4. AIに任せる範囲: 収集・差分抽出・仮説整理は任せ、判断と優先順位付けは人が担う。
5. 失敗パターン: ニュースだけを見る、CRM履歴だけを見る、業界テンプレに寄せすぎる。
6. 実務フロー: 15分で顧客情報を読み解く手順と、商談メモへの落とし込み。
7. まとめ: AI商談準備の価値は「速く調べる」ではなく「最初に見るべき情報を外さない」こと。

## 2. 共通ビジュアル方針

- トーン: BtoB向け、落ち着いた知的さ、フラット、シンプル、余白は十分だが外側の空白は残しすぎない。
- 色: 背景は白〜淡いグレー、差し色はネイビー、ブルー、ティール。警告や失敗例のみ淡いオレンジを使う。
- 形: 角丸カード、細い線、簡単なアイコン、矢印、ステップ番号。写真風、過度な3D、発光、複雑なグラデーションは避ける。
- 文字: サムネイル以外は画像内の文字を最小限にし、細かい日本語は生成後に編集で載せる前提にする。
- サイズ: noteサムネイルは 1280x670px 固定。記事内画像は 1200x675px を基本にし、note貼付後も横長で見やすくする。

## 4. 各画像の生成プロンプト

### No.0 noteサムネイル

用途: note一覧でタイトルを読ませ、営業責任者・マーケ責任者に「商談準備の話だ」と瞬時に伝える。

配置: note見出し画像。

生成プロンプト:

```text
1280x670pxのnote見出し画像。フラットでシンプルなBtoBビジネス向けデザイン。淡いグレー背景、左側に大きな日本語タイトル「AIで商談準備を速くするなら、最初に見るべき顧客情報は何か」を太字で読みやすく配置。タイトルは画面の約55〜65%を占め、文字間は詰めすぎない。右側に、顧客情報カード、AIの抽象アイコン、商談メモ、優先順位を示す3段のカードを配置。ネイビー、ブルー、ティールを差し色にする。過度な装飾、写真、3D、発光、細かすぎる文字、ロゴは不要。上下左右の外側余白は少なめ、ただしタイトル周辺の可読余白は十分に確保。
```

余白チェック方針:

- noteの中央トリミングを想定し、タイトルは左右60px・上下50px以内に寄せすぎない。
- 一覧縮小時にタイトル全文が読めない場合、背景要素を削り、タイトルを2〜3行に再配置する。
- 右側イラストの外側に広い空白が残る場合は、背景ごとトリミングせず、カード群を拡大して密度を上げる。

### No.1 記事内アイキャッチ

用途: 「商談準備は情報収集量ではなく、見る順番の設計」という主張を直感的に伝える。

配置: 導入直後。

生成プロンプト:

```text
1200x675px。フラットなビジネス図解イラスト。中央に営業担当者のデスクを抽象的に描き、左に大量の顧客情報カード、右に整理された商談準備メモを配置。中央にAIが情報カードを優先順位順に並べ替えている抽象アイコン。白〜淡いグレー背景、ネイビーとブルー中心、アクセントにティール。画像内テキストは最小限で、読めない細かい文字は入れない。シンプル、余白はあるが外側に無駄な空白を作らない。
```

余白チェック方針:

- 主役のカード群と商談メモが画面横幅の75%以上を使っているか確認する。
- 上下に余白が15%以上残る場合は、図解要素を拡大または上下中央へ寄せる。

### No.2 情報過多と見る順番の違い

用途: AI導入前後の差を「情報が増える」ではなく「順番が整う」として見せる。

配置: 問題提起後。

生成プロンプト:

```text
1200x675pxのフラット図解。左右比較レイアウト。左側は「情報が散らばっている状態」を表す、ばらばらのカード、ニュース、CRM、メール、Webページの抽象アイコン。右側は「見る順番が決まっている状態」を表す、1から5までの縦ステップカード。中央に細い矢印。日本語の細かい文字は入れず、カードには短い線やアイコンだけ。淡いグレー背景、左はやや薄い色、右はブルーとティールで整理された印象。シンプルで視認性重視。
```

余白チェック方針:

- 左右の比較領域が均等で、中央の矢印が小さくなりすぎていないか確認する。
- 図解の外周に背景だけの帯が残る場合、左右のカード群を拡大する。

### No.3 最初に見る顧客情報の優先順位

用途: 記事の核となる「何を見るべきか」を5段の階層で示す。

配置: 「最初に見る情報」章の冒頭または章末。

生成プロンプト:

```text
1200x675px。顧客情報の優先順位を示すフラットな階層図。上から順に5段の横長カードを階段状に配置。カードは、企業属性、直近の変化、接点履歴、課題仮説、意思決定構造を表すアイコン付き。ただし日本語文字は後から編集するため、生成時はアイコンと短いダミー線だけにする。背景は白、カードは淡いブルーグレー、重要度が高いカードほどネイビー寄り。左上に小さな見出しスペースを空ける。過度な装飾なし。
```

余白チェック方針:

- 5段カードが本文内で縮小されても見分けられる太さ・サイズか確認する。
- 見出しスペースを残す場合でも、上部余白が大きく見えたら後編集で見出しを入れて埋める。

後編集で載せる推奨ラベル:

- 1. 企業属性
- 2. 直近の変化
- 3. 接点履歴
- 4. 課題仮説
- 5. 意思決定構造

### No.4 AIと人の役割分担

用途: AI活用を過大評価せず、収集・整理と判断の境界を明確にする。

配置: 「AIに任せる範囲」章。

生成プロンプト:

```text
1200x675px。AIと人の役割分担を示すフラットな2カラム図解。左カラムにAIの抽象アイコンと情報収集、差分抽出、要約、仮説候補を表す4つの小カード。右カラムに人の判断を表す人物アイコンと、優先順位、問いの設計、商談で確認する論点を表す3つの小カード。中央に境界線と連携を示す細い矢印。文字は入れずアイコンと線で表現。背景は白、ネイビーとティール中心。落ち着いたBtoB資料風。
```

余白チェック方針:

- 2カラムの境界が中央にあり、左右どちらかが空きすぎないか確認する。
- 小カードが多くて小さく見える場合は、カード数を減らすか、アイコンを大きくする。

### No.5 失敗パターン比較

用途: 商談準備でありがちな偏りを、読者が自社の運用に照らして見られるようにする。

配置: 「失敗パターン」章。

生成プロンプト:

```text
1200x675px。商談準備の失敗パターンを3枚のカードで比較するフラット図解。カード1はニュースだけを見る、カード2はCRM履歴だけを見る、カード3は業界テンプレだけで考える状態を、抽象アイコンで表現。各カードの下に小さな警告マークと空欄メモ。日本語文字は生成せず、後から編集できる余白を各カード内に確保。背景は淡いグレー、警告色は淡いオレンジ、全体はネイビー基調。煽りすぎない落ち着いたトーン。
```

余白チェック方針:

- 3カードが横並びで均等に見えるか確認する。
- カード内の空欄が広すぎて未完成に見える場合、後編集で短いラベルを入れる。

後編集で載せる推奨ラベル:

- ニュース偏重
- CRM偏重
- 業界テンプレ偏重

### No.6 15分の商談準備フロー

用途: 記事後半で、抽象論を実務の時間配分に落とす。

配置: 「実務フロー」章。

生成プロンプト:

```text
1200x675px。15分で商談準備を進める流れを示す横長タイムライン図解。左から右へ5ステップのカードが並ぶ。各カードに時計、企業情報、変化、接点履歴、商談メモを表すシンプルなアイコン。上部に細いタイムライン、下部にメモ欄風の余白。文字は入れず、後から時間と短いラベルを追加できる構成。白背景、ブルーとティール中心、フラットでシンプル。
```

余白チェック方針:

- 5ステップが小さくなりすぎる場合は、上下2段ではなく横一列を維持しつつ各カード幅を広げる。
- 下部のメモ欄が広すぎる場合、タイムライン全体を下げずにカードを拡大する。

後編集で載せる推奨ラベル:

- 0-3分: 企業属性
- 3-6分: 直近変化
- 6-9分: 接点履歴
- 9-12分: 課題仮説
- 12-15分: 確認論点

### No.7 商談メモに残すべき出力

用途: 最後に、AIで整理した情報を商談準備の成果物へ変換する視点を示す。

配置: まとめ前。

生成プロンプト:

```text
1200x675px。商談準備メモの完成形を示すフラット図解。中央に大きな1枚の商談メモカードを配置し、周囲に顧客情報、変化、接点、課題仮説、確認質問の小さな入力カードが集まってくる構図。中央メモには3つの空欄セクションを作り、後から日本語ラベルを入れられるようにする。背景は白、線は細く、ネイビーとブルー、アクセントにティール。情報が整理されている印象。過度な装飾や小さな文字は不要。
```

余白チェック方針:

- 中央メモが画面の主役として十分大きいか確認する。
- 周辺カードが外側に寄りすぎてトリミングで切れそうな場合は、中央に寄せる。

後編集で載せる推奨ラベル:

- 商談で確認する問い
- 顧客側の変化
- 仮説と根拠

## 5. 生成後の確認・トリミング方針

1. 実寸確認: サムネイルは必ず 1280x670px、記事内画像は 1200x675px を基準に書き出す。
2. 外側余白確認: 四辺の背景だけの帯が全体幅または高さの10〜12%を超える場合はトリミングまたは要素拡大を行う。
3. 可読性確認: サムネイルは幅320px相当に縮小してもタイトルの主語と論点が読めることを確認する。
4. 本文内視認性確認: 図解は幅700px相当に縮小しても、カード数、矢印、主役要素が判別できることを確認する。
5. noteトリミング確認: サムネイルは中央基準で表示されてもタイトル・主要図形が欠けないよう、重要要素を安全域内に収める。
6. 文字崩れ確認: 画像生成で日本語が崩れた場合、図中文字は削除し、Canva等またはローカル編集で日本語ラベルを載せ直す。
7. 統一感確認: 全画像で背景色、線幅、角丸、差し色が大きくずれていないか確認し、違和感がある画像は再生成する。

## 6. 制作順序

1. 本文構成と最終タイトルを確定する。
2. No.1〜No.7 の記事内画像を生成し、章ごとの論点に合うか確認する。
3. 図解内の日本語ラベルを後編集で整える。
4. 各画像の外側余白と本文内縮小時の視認性を確認する。
5. 最後に No.0 のnoteサムネイルを生成し、最終タイトルの可読性を最優先で調整する。
